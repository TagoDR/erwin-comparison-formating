# Erwin Compare Formatter

A specialized utility to transform Erwin Data Modeler difference reports into a modern, filterable, and easy-to-analyze visual interface.

---

## 📂 For Users

### How to get the application

The application is distributed as a single HTML file that works entirely offline.

- **Download:** [Click here to access the project folder and download the `Erwin_Formatter_CompleteCompare.html` file](./dist/Erwin_Formatter_CompleteCompare.html) (or locate it in the `dist/` folder of this repository).

### Usage Instructions

1.  **Open the file:** Double-click the downloaded `.html` file to open it in your preferred browser (Chrome, Edge, Firefox, etc.).
2.  **Load the report:**
    - Drag and drop the HTML file generated by **Erwin "Complete Compare"** into the central area.
    - Or click the selection area to browse for the file on your computer.
3.  **Analyze the data:**
    - Use the **Filters** at the top to isolate additions, changes, or deletions.
    - View the **Hierarchy** of objects (Tables -> Columns -> Properties) with clear indentation.
    - **Pastel Colors:** Green (Addition), Purple (Change), and Red (Deletion).
    - **Copy Data:** Use the copy buttons (clipboard icons) to copy individual table names or the full list of impacted tables.

## 🛠️ Development and Maintenance

This section is for developers who want to modify or build the project.

### Technical Stack

- **Framework:** [Lit](https://lit.dev/) (Web Components) for a reactive and lightweight interface.
- **Bundler:** [Vite](https://vitejs.dev/) with the `vite-plugin-singlefile` plugin to ensure all CSS, JS, and assets are embedded into a single HTML file.
- **UI/Design:** Based on the **Bootflat** framework (Dark Slate theme) with customizations for data visualization.
- **Icons:** [Tabler Icons](https://tabler-icons.io/) loaded via `vite-svg-loader` and injected as Lit components.
- **Linting:** [Biome](https://biomejs.dev/) for code standardization.

### Main Commands

1.  **Installation:**
    ```bash
    npm install
    ```
2.  **Development Environment:**
    ```bash
    npm run dev
    ```
3.  **Build:**
    Generates the final file in the `dist/` folder.
    ```bash
    npm run build
    ```
    _Note: The build script uses PowerShell to automatically rename the output file._
