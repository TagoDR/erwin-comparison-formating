# Project: Erwin Compare Formatter

## 1. Overview
The **Erwin Compare Formatter** is a specialized utility designed to ingest HTML difference reports generated by Erwin Data Modeler. It transforms the raw comparison data into a highly visual, searchable, and filterable interface optimized for human analysts.

The final output is a single-file HTML application (built via Vite) for easy distribution and offline use.

## 2. Technical Stack
- **Framework:** Vite with TypeScript
- **State management:** [Nanostores](https://github.com/nanostores/nanostores) for [Lit](https://lit.dev/)
- **Output Type:** Single-file HTML (using `vite-plugin-singlefile`)
- **UI Framework:** [PureCSS](https://purecss.io/) with custom dark theme.
- **Target Language:** Portuguese (pt_BR) for user interface instructions.
- **Linting:** Biome

## 3. UI/UX Requirements

### 3.1. Layout
The interface is divided into two primary sections:

1. **Header (Static Top Bar):**
    - A slim, high-contrast row at the top of the screen.
    - **File Loading:** Area to choose a file via standard input or drag-and-drop.
    - **File Management:** A button to "Fechar Arquivo" (Close File) which clears the current session.
    - **Filters:** (Planned) Selects for Change Type and Object Category.

2. **Main Display (Split Layout):**
    - **Stats Panel:** A summary table with 3 rows (Geral, Tabelas, Colunas) and 5 columns (Tipo, Total, Inclusão, Alteração, Exclusão).
    - **Data Table Panel:** The core enhanced data display as defined in section 3.3.

### 3.2. Data Processing (Parser Logic)
- **Indentation Parsing:** Interpret leading spaces in the Erwin "Type" column to build an object hierarchy.
- **Status Hoisting:**
    - Property-level changes (I/A/E) are hoisted to the parent object (e.g., Table or Column).
    - Objects inherit the most significant change status of their children/properties.
- **View Flags:** "Logical Only" and "Physical Only" properties are hoisted to the identifier row.

### 3.3. Table Configuration
| Column | Description |
| :--- | :--- |
| **Type** | Indented object hierarchy (mono-font). |
| **Prop** | Parent object identifier. |
| **Change** | I (Inclusion), A (Alteration), or E (Exclusion). |
| **View** | L (Logical), P (Physical), or L/P. |
| **Left Model** | Working model data. |
| **Right Model** | Baseline model data. |

## 4. Visual Coding (Color Scheme)
- **Table:** Dark Green (New), Dark Purple (Changed), Dark Red (Deleted).
- **Column:** Green (New), Purple (Changed), Red (Deleted).
- **Other Objects:** Orange (with decreasing intensity per nesting level).

## 5. File Structure
- `src/parser/`: Erwin HTML processing logic.
- `src/components/`: PureCSS-based Lit components (`app-header`, `app-stats`, `app-table`).
- `src/store/`: State management for data, filtering, and loading states.
